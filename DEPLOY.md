## 1. Deploy the backend to Render

### 1a. Create the service

1. Push your repo to GitHub if not already.
2. Render dashboard â†’ **New +** â†’ **Blueprint**.
3. Connect the repo. Render reads `furnotech_ecommerce/backend/render.yaml`
   and asks you to fill in the secret env vars.

### 1b. Fill in the secret env vars

These are flagged `sync: false` in `render.yaml` â€” Render will prompt for
each one on the create-service screen:

| Key                       | Value                                                              |
| ------------------------- | ------------------------------------------------------------------ |
| `DATABASE_URL`            | `mysql://furnotech:ecommerce%4012345@68.178.232.132:3306/ecommerce` (URL-encode `@` as `%40` in the password) |
| `CORS_ALLOWED_ORIGINS`    | Leave blank for now â€” we'll fill it after Vercel deploys           |
| `CLOUDINARY_CLOUD_NAME`   | From your Cloudinary dashboard (or blank to disable image upload)  |
| `CLOUDINARY_API_KEY`      | "                                                                  |
| `CLOUDINARY_API_SECRET`   | "                                                                  |
| `RAZORPAY_KEY_ID`         | Blank to use simulated payments                                    |
| `RAZORPAY_KEY_SECRET`     | Blank to use simulated payments                                    |
| `RAZORPAY_WEBHOOK_SECRET` | Blank                                                              |
| `ERP_API_URL`             | Blank â€” service auto-falls-back to internal `INT-â€¦` IDs            |
| `ERP_API_KEY`             | Blank                                                              |
| `EMAIL_HOST`              | Blank (uses console backend) or your SMTP host                     |
| `EMAIL_HOST_USER`         | Your SMTP user (only if `EMAIL_HOST` set)                          |
| `EMAIL_HOST_PASSWORD`     | Your SMTP password                                                 |
| `FRONTEND_URL`            | `https://<your-project>.vercel.app` â€” set after step 2             |

`SECRET_KEY` is auto-generated by Render (`generateValue: true`). `DEBUG` is
forced to `False`. `ALLOWED_HOSTS` already includes `.onrender.com`.

### 1c. First deploy

Render runs `./build.sh` which:
- `pip install -r requirements.txt`
- `collectstatic` (admin assets served by WhiteNoise)
- `migrate` (applies any pending Django migrations to GoDaddy MariaDB)

Then starts gunicorn. Watch the deploy logs â€” first build takes ~3-4 minutes.

### 1d. Verify it's up

Render gives you a URL like `https://furnishop-backend.onrender.com`.

```bash
curl https://furnishop-backend.onrender.com/api/products/
```

Should return JSON with a paginated product list (200 OK). If you get 500,
check Render â†’ Logs for the stack trace â€” most likely a missing env var.

> **Cold-start note:** Free plan spins the service down after 15 min of
> inactivity. First request after sleep takes ~30s to wake. Acceptable for
> dev; upgrade to Starter ($7/mo) for production traffic.

## 2. Deploy the frontend to Vercel

### 2a. Import the project

1. Vercel dashboard â†’ **Add New** â†’ **Project**.
2. Pick the same GitHub repo.
3. **Root Directory** â†’ `furnotech_ecommerce/frontend`.
4. Vercel auto-detects Vite (because `vercel.json` says so). Leave other
   settings as defaults.

### 2b. Set the env var

In **Project Settings â†’ Environment Variables** add:

| Key                   | Value                                                       |
| --------------------- | ----------------------------------------------------------- |
| `VITE_API_BASE_URL`   | `https://<your-render-service>.onrender.com/api`            |
| `VITE_RAZORPAY_ENABLED` | `false` (or `true` once Razorpay keys are wired)          |

Apply to **Production**, **Preview**, and **Development** so PR previews work.

### 2c. Deploy

Vercel builds: `npm ci` â†’ `npm run build` â†’ serves `dist/` from the edge CDN.
Build takes ~30s. You get a URL like `https://furnishop-xyz.vercel.app`.

## 3. Tell the backend about the frontend

Open Render â†’ your service â†’ **Environment** â†’ set:

| Key                    | Value                                       |
| ---------------------- | ------------------------------------------- |
| `CORS_ALLOWED_ORIGINS` | `https://<your-project>.vercel.app`         |
| `FRONTEND_URL`         | `https://<your-project>.vercel.app`         |

Render restarts the service automatically. Done â€” open the Vercel URL, you
should see the storefront pulling products from Render.

> Vercel PR previews (`<branch>-<hash>.vercel.app`) are already allowed via
> the `CORS_ALLOWED_ORIGIN_REGEXES` rule in settings.py â€” no extra config.

## 4. Create the first admin user

SSH into Render isn't on the free tier, so use the one-off **Shell** tab on
the service dashboard (Free plan unlocks 15 min of shell per month).

```bash
python manage.py createsuperuser
```

Or seed a dev admin via the existing management command:

```bash
python manage.py seed_admin
```

(That command creates `dev-admin@furnishop.local` / `dev-user@furnishop.local` /
`dev-dealer@furnishop.local` â€” change the passwords immediately for prod.)

  -d '{"email":"<your-admin>","password":"<your-pwd>"}'

On the live frontend:
1. Sign in via `/login`.
2. Browse `/`, add a product to cart, complete checkout (simulated payment).
3. Check `/admin-dashboard` â†’ Overview should show your test order.

## 6. After the smoke test

- **Custom domain.** Vercel â†’ Domains â†’ add your domain. Vercel auto-issues
  a Let's Encrypt cert. After it propagates, add the new domain to
  `CORS_ALLOWED_ORIGINS` on Render.
- **Razorpay live keys.** Set `VITE_RAZORPAY_ENABLED=true` on Vercel, fill
  `RAZORPAY_KEY_ID` / `KEY_SECRET` / `WEBHOOK_SECRET` on Render. Razorpay
  webhook URL is `https://<render>.onrender.com/api/payment/webhook/`.
- **Real ERP integration.** Drop the real ERP URL into `ERP_API_URL` and
  the order-sync service starts using it instead of internal `INT-â€¦` IDs.
- **Email.** Gmail SMTP works: `EMAIL_HOST=smtp.gmail.com`,
  `EMAIL_PORT=587`, `EMAIL_USE_TLS=True`, and an *app password* (not your
  Gmail password) for `EMAIL_HOST_PASSWORD`. SendGrid / Postmark / SES
  similarly work â€” just swap in the right host + credentials.
- **Render Starter plan.** Bump to $7/mo to drop cold-start and get
  zero-downtime deploys.

## Troubleshooting

| Symptom                                | Where to look                                                                            |
| -------------------------------------- | ---------------------------------------------------------------------------------------- |
| `502 Bad Gateway` on Render            | Logs tab â€” gunicorn probably failed to import. Usually a missing env var or a typo.      |
| `CORS error` in browser console        | Check the failing request's origin matches one in `CORS_ALLOWED_ORIGINS` (Render env).   |
| `DisallowedHost` in logs               | Add the host to `ALLOWED_HOSTS` (Render env). `.onrender.com` is auto-allowed.           |
| `relation does not exist` on first hit | Migrations didn't run. SSH in â†’ `python manage.py migrate`. Re-check `build.sh` output.  |
| `Database returned invalid datetime`   | The DB-side timezone fix is already in `settings.py`; restart the service if you migrated old data. |
| Login works, then I get bounced        | Means the refresh token endpoint is failing. Check `/api/auth/token/refresh/` returns 200 with `{access, refresh}`. |
| Frontend `404` on direct-load of route | `vercel.json` SPA rewrite is missing or wrong root dir â€” check it points at `furnotech_ecommerce/frontend`. |
